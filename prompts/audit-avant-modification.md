# Prompt standard — Audit avant modification FIPLA

Tu travailles sur le projet FIPLA Dashboard.

## Mission
Avant toute modification, analyser la demande, identifier les fichiers concernés, repérer les zones sensibles et proposer une stratégie minimale de modification.

## Interdictions absolues
- ne pas modifier immédiatement
- ne pas refactorer
- ne pas toucher aux calculs fiscaux
- ne pas toucher à la connexion API TaxWare
- ne pas toucher au PDF sans instruction explicite

## Méthode attendue
1. Reformuler la demande
2. Identifier les fichiers probablement concernés
3. Identifier les zones sensibles à protéger
4. Décrire les risques éventuels
5. Proposer la stratégie minimale
6. Attendre ensuite la phase de modification

## Format de réponse obligatoire
Toujours répondre avec cette structure :
- état d’avancement
- demande reformulée
- fichiers potentiellement concernés
- zones sensibles à protéger
- risques identifiés
- stratégie minimale recommandée
- mention finale : tâche terminée
